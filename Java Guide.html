1.Lambda表达式（Lambda Expressions）
  • Lambda 允许把函数作为一个方法的参数（就是把函数作为参数传递进方法中）
基础语法
  • 格式
(parameters) -> expression
或
(parameters) ->{ statements; }
  • 案例
// 1. 不需要参数,返回值为 5  
() -> 5  
  
// 2. 接收一个参数(数字类型),返回其2倍的值  
x -> 2 * x  
  
// 3. 接受2个参数(数字),并返回他们的差值  
(x, y) -> x – y  
  
// 4. 接收2个int型整数,返回他们的和  
(int x, int y) -> x + y  
  
// 5. 接受一个 string 对象,并在控制台打印,不返回任何值(看起来像是返回void)  
(String s) -> System.out.print(s)
常见场景：
多线程（Runable接口）

3.接口默认方法（Default methods ）
  • 新增了接口的默认方法
  • 简单说，默认方法就是接口可以有实现方法，而且不需要实现类去实现其方法。
  • 我们只需在方法名前面加个 default 关键字即可实现默认方法。
代码示例
public interface Vehicle {
   default void print(){
      System.out.println("我是一辆车!");
   }
}
其他：Optional 类（解决空指针、搭配Lambda使用更舒服！）
  • Optional 类是一个可以为null的容器对象
  • Optional 类的引入很好的解决空指针异常。
代码示例
主要避免空指针，例如下面这段
    public static String getGender(Student student){
        if(null == student){
            return "Unkown";
        }
        return student.getGender();
    }
优化后：Optional类结合lambda表达式的使用能够让开发出的代码更简洁和优雅
    public static String getGender(Student student){
       return Optional.ofNullable(student).map(u -> u.getGender()).orElse("Unkown");
    }
举例2
        String s = null;
        // if语句
        if (s == null) {
            System.out.println("Unkown");
        } else {
            System.out.println(s);
        }
        // Optional类
        String unkown = Optional.ofNullable(s).map(u -> u.toLowerCase()).orElse("Unkown");
        System.out.println(unkown);

HTTP Client API（java.net.http模块java.net.http包）
  • 定义HTTP客户端和WebSocket API（就是http请求呗）
  • 提供HTTP（版本1.1和2）的高级客户端接口和WebSocket的低级客户端接口。 定义的主要类型是：  HttpClient   HttpRequest   HttpResponse   WebSocket
示例代码
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;

public class T {
    public static void main(String[] args) {
        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder().uri(URI.create("http://foo.com/")).build();
        client.sendAsync(request, HttpResponse.BodyHandlers.ofString()).thenApply(HttpResponse::body).thenAccept(System.out::println).join();
    }
}

视图（View）
MySQL 视图（View）是一种虚拟存在的表，同真实表一样，视图也由列和行构成，但视图并不实际存在于数据库中。行和列的数据来自于定义视图的查询中所使用的表，并且还是在使用视图时动态生成的。
  • 简单来说，使用的时候把视图当成一张表就行，视图里面就是查询语句。
# 创建视图
CREATE VIEW <视图名> AS <SELECT语句>
# 创建视图
CREATE VIEW v_staffs AS SELECT * FROM staffs;
# 更新视图，会同步更新基本表数据
UPDATE v_staffs SET salary = salary + 1000 WHERE uid = 1;
# 删除视图
DROP VIEW v_staffs;

左连接（Left Join）
  • LEFT JOIN（左连接）：获取左表所有记录，即使右表没有对应匹配的记录。
  • 最常用的就是左连接，因为我们看书也是从左往右，所以习惯用左关联。
# 左连接
SELECT column_name(s)
FROM table1
LEFT JOIN table2
ON table1.column_name=table2.column_name;
# 作用同上（某些数据库中也称左连接为：左外连接）
SELECT column_name(s)
FROM table1
LEFT OUTER JOIN table2
ON table1.column_name=table2.column_name;
举例
# 左连接
SELECT *
FROM sys_user
LEFT JOIN sys_dept
ON sys_user.dept_id = sys_dept.dept_id
# 作用同上
SELECT *
FROM sys_user
LEFT OUTER JOIN sys_dept
ON sys_user.dept_id = sys_dept.dept_id
右连接（Right Join）
  • RIGHT JOIN（右连接）： 与 LEFT JOIN 相反，用于获取右表所有记录，即使左表没有对应匹配的记录，则结果为 NULL。
# 右连接
SELECT column_name(s)
FROM table1
RIGHT JOIN table2
ON table1.column_name=table2.column_name;
# 同上
SELECT column_name(s)
FROM table1
RIGHT OUTER JOIN table2
ON table1.column_name=table2.column_name;
举例
# 右连接
SELECT *
FROM sys_user
right JOIN sys_dept
ON sys_user.dept_id = sys_dept.dept_id
# 同上
SELECT *
FROM sys_user
right OUTER join sys_dept
ON sys_user.dept_id = sys_dept.dept_id
内连接（Inner Join）
  • INNER JOIN（内连接,或等值连接）：获取两个表中字段匹配关系的记录。
  • INNER JOIN 关键字在表中存在至少一个匹配时返回行。
# 内连接
SELECT column_name(s)
FROM table1
INNER JOIN table2
ON table1.column_name=table2.column_name;
# 同上
SELECT column_name(s)
FROM table1
JOIN table2
ON table1.column_name=table2.column_name;
举例
# 右连接
SELECT *
FROM sys_user
INNER  JOIN sys_dept
ON sys_user.dept_id = sys_dept.dept_id
# 同上
SELECT *
FROM sys_user
join sys_dept
ON sys_user.dept_id = sys_dept.dept_id
全连接（Full Join）
  • 只要左表（table1）和右表（table2）其中一个表中存在匹配，则返回行.
  • 结合了 LEFT JOIN 和 RIGHT JOIN 的结果。
  • 在MySQL中没有全连接，不过可以使用联盟查询（UNION）
SELECT column_name(s)
FROM table1
FULL OUTER JOIN table2
ON table1.column_name=table2.column_name;
# 同上
SELECT column_name(s)
FROM table1
FULL OUTER JOIN table2
ON table1.column_name=table2.column_name;
联盟查询（UNION）
  • UNION 操作符查询两张表中的数据（仅仅查出不同的值）
SELECT column_name(s) FROM table1
UNION
SELECT column_name(s) FROM table2;
  • UNION 操作符查询两张表中的数据（可以查出相同的值）
SELECT column_name(s) FROM table1
UNION ALL
SELECT column_name(s) FROM table2;
举例
# 仅仅查出不同的值
SELECT * FROM sys_user
union
select * from sys_user;
# 可以查出相同的值
SELECT * FROM sys_user
union all
select * from sys_user;
